# UIモード切り替え設計書
**作成日**: 2026-06-11  
**ブランチ**: feature/ui-mode-switching

---

## 背景・問題

アプリが機能を拡充するにつれ、ヘッダーツールバーに13以上のボタンが並ぶようになった。  
アイコンのみのボタン（BarChart3 / Compass / Mic2 など）はツールチップが出るまで機能が不明で、  
アニメ視聴が趣味程度のライトユーザーには「何をするアプリなのか」がわかりにくい。

---

## ペルソナ議論まとめ

### ペルソナ A — ライトユーザー「田中さん」
- 30代会社員。アニメは好きだが追いすぎない。
- 今期の放映作品を把握して「見たい」「視聴中」「完了」を管理したい。
- ティア表のドラッグ操作、PNG出力、共有URLなどは「難しそう」で使わない。
- アイコンだけのボタンが並ぶと「押して大丈夫か」と躊躇する。
- **ニーズ**: アニメの一覧 + 視聴ステータス管理 + 契約サブスクとの照合

### ペルソナ B — コアユーザー「nobu」
- 毎クール全作品チェック。ティア表をSNSに投稿する。
- 声優ページ・過去作探索・サブスク診断・分析ダッシュボードを活用する。
- 操作密度が高い方が好みで、コンパクトなアイコンUIを好む。
- **ニーズ**: 全機能フルアクセス + 年/期切り替え + 高度なフィルタ

### 解決策の方向性
アイコン問題の「根本解決」は2段階:
1. **ショートタームト**: UIモードを切り替えて、ライトユーザーには不要な機能を隠す
2. **ロングターム**: ライトモードではさらにテキスト説明を追加した導線（tooltipなど）

## 実装方針

### ストレージ
- `localStorage` キー: `anime-tier-board:uiMode`
- 値: `"simple"` | `"pro"`
- **デフォルト**: `"simple"`（初回訪問者はライトモードから入る）

### コンテキスト
- `lib/ui-mode.tsx` に `UiModeContext` + `UiModeProvider` + `useUiMode` hook を定義
- `app/providers.tsx` でラップ

### 切り替えUI
- `/settings` ページに「表示モード」セクションを追加
- シンプル ↔ プロ をラジオボタン or トグルで選択
- ヘッダーにも小さな「⚙ 設定」リンク（プロモードでは現行の icon-button を維持）

### 影響するコンポーネント
1. `components/MobileNav.tsx` — ライト時は3項目のみ
2. `components/TierBoardApp.tsx` — ライト時はツールバーを最小化
3. `app/settings/settings-client.tsx` — モード切り替えUIを追加
4. `app/providers.tsx` — UiModeProvider を追加
