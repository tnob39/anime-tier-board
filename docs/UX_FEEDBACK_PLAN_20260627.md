# UX Feedback Plan — 2026-06-27（オーナーブレインダンプ整備版）

> オーナー音声指示（2026-06-27）を構造化したフィードバックパケット。
> `docs/UX_DIRECTION.md` を本確定まで上書きしない（CLAUDE.md の運用ルールに従う）。
> 実装は **Grok 委任** 前提で、スライス（issue）単位に分解する。

## 0. 全体方針（このパケットの背骨）

1. **ホームの主役を「マイリスト（視聴管理）」にする。** 現状の watchlist V2(Grok版) のレイアウト・スペース感を
   ホームに持ち込む。下方に放映カレンダー。
2. **Grok 版 watchlist を最終版として全ユーザーにリリース。** Codex版・オーナー限定の3択モード切替は撤去。
3. **オーナー限定ビュー / 一般ビューの二層化を明確化。**（確定 2026-06-27）
   - 一般ビュー ナビ: **ホーム(=マイリスト) / Tier / 分析** の3本。**探すタブなし。** 分析はサブスクカバー率のみ。
   - オーナー限定ビュー ナビ: 上記 + **さがす**（4本）。分析に「好み分析ダッシュボード（アコーディオン）」追加。
   - ＝ナビは一般/オーナーで共通、差分は「さがす」タブの有無のみ。
4. **テーマ（ダーク/ライト）のヘッダー・フッター適用バグを修正。**

ゲート機構は既存 `lib/owner.ts`（`OWNER_EMAIL = tnob38@gmail.com` / `isOwnerEmail()`）を流用。

## 1. ホーム = マイリスト化（最重要・構造変更）

**現状**: ホーム(`app/page.tsx`) は独自レイアウト。マイリストは `app/watchlist/`（V2 Grok版あり）。

**要件**（確定 2026-06-27: ホームを**マイリストで置き換え**。既存ホームの機能で残す価値があるものは移植を検討）:
- ホームの主コンテンツを **マイリスト（Grok版 watchlist のレイアウト）で置き換える**。
- 既存ホーム(`app/page.tsx`) の独自要素（おすすめ/今期注目など）は、価値あるものだけマイリスト側へ移植。
  実装時に「移植 / 廃止」を1要素ずつ判断し PR 説明に明記。
- 期セクション（今期 / 来期 / その他）を維持し、各セクションは横スライダー。
- **「全何話」表示を全面採用**（各カードに総話数 / 視聴進捗を出す）。Grok版で既に出ている表現を標準化。
- **曜日の概念をマイリスト側に付与**（カードに放映曜日を出す / 曜日でまとまりが分かる）。
- ホーム **下方に放映カレンダー** を残す。ただしカレンダーの配置・デザイン・余白（スペース）は
  マイリストのカード/レーンの作法に揃える（独自デザインをやめてマイリスト基準に統一）。

**非対象 / 保留**:
- 「シェア（共有受信）側に視聴中/予定が反映されるか」→ オーナーが「なんでもない」と取消。**対応不要。**

**受け入れ条件**:
- 一般ユーザーでホームを開くと、今期/来期/その他の横スライダー＋全話数＋曜日が見える。
- ホーム下方にカレンダーがあり、マイリストのカード作法と視覚的に揃っている。

## 2. Grok 版 watchlist を最終リリース（一般採用）

**現状**: `app/watchlist/` に V1 / V2(codex) / V2(grok) が併存。`watchlist-switch.tsx` でオーナー限定3択切替。
`watchlist-v2-grok.css` は独自ダーク色 `--wl2g-*` を使用。

**要件**:
- **Grok版を全ユーザーの本採用**にする。Codex版 (`watchlist-client-v2-codex.tsx` / `watchlist-v2-codex.css`)
  と3択モード切替 (`watchlist-switch.tsx`)・オーナーゲート(`canPreview`)を撤去。
- Grok版独自ダーク色 `--wl2g-*` を **アプリ共通テーマトークン**（`--background`/`--surface`/`--ink`/`--line` 等）に載せ替え、
  テーマ切替に追従させる。
- 旧 V1 watchlist の扱い: ホーム化（§1）後に未使用になるなら削除候補（実装時に参照確認）。

**受け入れ条件**:
- 一般ユーザーに Grok版レイアウトが出る。モード切替UIが本番から消える。
- watchlist のダーク表示がテーマトークン由来になり破綻しない。

## 3. 追加UIの刷新（今期/来期リストから横スライド追加）

**現状**: ホーム追加セクション or explore からの追加導線。オーナー評「いけてない」。

**要件**:
- 「今期リスト」「来期リスト」のようなまとまりから、**横スライド式**で作品を選んでマイリストに追加できるUIにする。
- 1タップ追加（ステータス付与）まで横スライダー内で完結。

**受け入れ条件**:
- マイリスト（ホーム）上で、今期/来期の候補を横スライドしながら追加できる。

## 4. 分析タブのシンプル化 ＋ オーナー/一般ビュー分岐

**現状**: `app/dashboard/`（分析）にサブスク分析＋アニメタイプ/ジャンル/声優分析が同居。

**要件（一般ビュー）**:
- **サブスクのカバー率だけ**をシンプルに見せる（上部セクション）。
- アニメ傾向・声優傾向・ジャンル傾向は **非表示**。
- **声優リンクは一旦なし**。

**要件（オーナー限定ビュー）**:
- 上部に **サブスク（カバー率）セクション**。
- その下に **「好み分析ダッシュボード」をアコーディオン**で配置。開くとアニメタイプ / ジャンル / 声優分析が入る。
  - つまり既存の好み分析系は撤去せず、アコーディオンの中に格納してデフォルト閉じる。
  - 声優リンクはこのアコーディオン内でも一旦なし（リンクのみ除去、傾向表示は残してよい）。

**受け入れ条件**:
- 一般ユーザーの分析タブ = サブスクカバー率のみ。
- オーナーの分析タブ = サブスク + 好み分析アコーディオン（中にアニメタイプ/ジャンル/声優）。

- **オーナー限定で「探す」を表示**。

**受け入れ条件**:
- 一般ユーザーのナビに「探す」がない。オーナーには出る。
- 既存の年代/人気ビューは現状維持。
