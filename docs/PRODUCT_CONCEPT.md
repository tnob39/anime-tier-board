## 1. ひとことコンセプト

> **numanie（ぬまにえ）= 好きなアニメを記録して、もっと沼に。**
> 季節アニメの「見る・記録する・振り返る・最適化する」を**ひとつに束ねる、自分専用のアニメ視聴ベース**。

- 由来: 「沼に（ハマる）」＋ フランス語 *manie*（熱中・偏愛）。
- タグライン候補: 「アニメ視聴を、自分のものに。」

competitors が機能ごとに分かれているのに対し、numanie は**1つで完結**するのが軸。

|------|---------|--------|----------|-------------|

## 3. ターゲット — 2ペルソナを1アプリで両立（確定）

ライト層とヘビー層の**どちらも切り捨てず、単一の IA 上で両立**させる。
ナビやルートをペルソナ別に分岐しない（旧 simple/pro モード分岐は**履歴**。現行は 5 タブ単一 IA + 表示設定）。

### ペルソナA：ライト層「つん」
- **属性**: 20代後半・会社員。今期は2〜3本を"ながら見"。配信サブスクは1〜2個加入済み。
- **ゴール**: 見逃さず追いたい。今夜・週末に「何を見ればいいか」をすぐ知りたい。
- **不満（既存ツールへの）**: 記録が面倒、機能が多すぎて使いこなせない、開発者向けの情報が邪魔。
- **使う機能**: 視聴ステータス（見たい/視聴中/完了）・今夜何見る・放送カレンダー・サブスクで見られるか。
- **使わない／後回しにしやすい機能**: Tier細分化・PNG出力・声優分析・無制限カスタム。

### ペルソナB：ヘビー層「ぬま」
- **属性**: 30代・沼の住人。今期10本以上。声優・制作会社まで追う。複数サブスク。SNSで語りたい。
- **ゴール**: 見た作品を**評価・序列化して資産として残す**。傾向を振り返る。布教する。
- **不満**: 記録の表現力が足りない、Tierや評価を作り込めない、共有が映えない。
- **使う機能**: Tier表（D&D・PNG）・お気に入り度・メモ・視聴リズム・好み分析・声優・布教カード・共有。
- **重視**: 情報密度・カスタマイズ性・一覧性。

### 設計の含意（現行）
- **つん**向けの主役は「今/今週の視聴サポート」。ホームの一次情報をそこに寄せ、道具を減らし迷わせない。
- **ぬま**向けの主役は「記録・評価・分析」。Tier・詳細編集・振り返りへ深く辿れる導線を残す。
- **IA は単一**（5 タブ）。ペルソナ差は「モード切替で別アプリ化する」のではなく、**優先表示・密度・二次操作の畳み方**で吸収する。
- **Visual / Simple** は同一機能の**表示設定**（画像の有無・レイアウト密度）。IA や機能セットの分岐ではない。Simple は画像 HTTP を出さず空の画像枠も残さない。詳細は [UIUX_RECONSIDERATION_20260711.md](./UIUX_RECONSIDERATION_20260711.md)。

### 視聴スタイル軸（ペルソナと直交する第2の軸）— 2026-06-13 追加

ライト/ヘビーとは**独立した軸**として「いつ・どう見るか」が存在する。これがホームの主役選びに強く効く。

| | リアタイ派 | サブスク派 |
|---|---|---|
| いつ見る | 放送日その日（TV） | 放送日の**翌日以降**、自分の都合で（配信） |
| 「今夜放送」の価値 | 高い | **薄い**（その日には見ない） |
| 本当に欲しい情報 | 今夜の放送 | **今 配信で見られる未視聴話** / いつ配信されるか |

- ✅ **確定（2026-06-14, オーナー）: サブスク派が主軸**。リアタイ派は副次対応。numanie は配信サービス連携が機能の中心であることと整合。
- ペルソナ議論でも、つん（「dアニメで見られるか」）・ぬま（「未視聴最新話」）とも**サブスク派的**で、「今夜放送をその日に」を強く求めた人はいなかった。
- → **設計含意: ホームの主役は「今夜放映」より「配信済み・未視聴のキャッチアップ」に寄せる**。今夜放送はリアタイ派向けに残すが副次。
- 詳細と2×2マトリクスは [HOME_DESIGN_OPTIONS.md](./HOME_DESIGN_OPTIONS.md) §1.5。

## 4. 機能インベントリ（実装済みの棚卸し）

> 画面構成を再設計する際の「手持ちのカード」一覧。v0.1〜v1.6 実装済み。

**① 記録・整理（コア）**
- Tier表：S/A/B/C/D、ドラッグ&ドロップ、PNG出力、共有
- 視聴ステータス：見たい/視聴中/完了/一時停止/中止（5段階・チップUI）
- 視聴管理：お気に入り度（★5）、いつ見る、メモ、視聴リズム（毎週/まとめて/ゆっくり）

**② 視聴サポート（今/今週）**
- 今夜何見る？：続きを見る / 今夜完結したい（候補提示・理由タグ）
- 放送カレンダー：曜日別（視聴中・見たいのみ）
- PWA：ホーム追加・App Shortcuts・バッジ（今日の本数）・プッシュ通知

**③ 発見・探索**
- explore：季節アニメ、ジャンル/フォーマット/視聴可否フィルタ、「今すぐ見放題」
- 生涯ランキング（計画・Issue #74）、声優ページ

**④ コスパ最適化**
- サブスク最適化：見放題カバー率、サービス別カバー作品、診断ウィザード
- オンボーディング：初回サブスク登録

**⑤ ソーシャル**
- 共有：Tier / 分析 / 視聴リスト の公開URL
- コメント・絵文字リアクション、布教カード（おすすめコメント付き）

**⑥ 分析**
- 好み分析：視聴ステータス分布、ジャンル/制作会社/声優の傾向（グラフ）

**⑦ 基盤**
- Googleログイン（NextAuth）、表示設定（Visual / Simple）、共有ページ専用シェル
  - ※旧「シンプル/プロモード」による IA 分岐は廃止済み。現行の表示設定は機能差分ではなく見た目の切替。

## 5. 目指す顧客体験（原則）

UX_DIRECTION.md の原則（Letterboxd / AniList / Crunchyroll / Spotify 基準）を継承し、コンセプト視点で補強する：

1. **ホームは「自分のもの」が映る**。開いた瞬間に "自分の今" が分かる。
2. **単一 IA の上で、文脈ごとの次アクションを優先する**。「モードで主役を変える」のではなく、画面・状態に応じた「次にやること」を前面に出す。ライト向けには「今/今週」の視聴サポートを、ヘビー向けの記録・評価・分析は辿れる深さとして残す（ナビ分岐はしない）。
3. **1画面1主アクション**。道具箱を全部見せない。二次操作は畳む。
4. **横スクロールは意味のある所だけ**（時系列＝カレンダー等）。基本は縦リストで俯瞰。
5. **メタ情報を見せない**（データソース名・キャッシュ・件数デバッグ）。
6. **登録は後**。価値を見せてからログインを促す（ゲスト閲覧許容）。
7. **オールインワンでも迷わせない**。機能の多さを、IAと役割分担で隠す。
