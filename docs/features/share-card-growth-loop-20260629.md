# シェアカード成長ループ — 仕様検討（リアクション撤去 / コメント / 「自分のシェアカードを作る」CTA）

> **位置づけ**: シェアページ（`/share/[shareId]` Tier表シェア・`/share/season/[shareId]` 期まとめシェア）を「獲得ファネルの最上流」と捉え直し、social UI を再設計する。
> **上位参照**: `PRODUCT_CONCEPT.md`（特に §3 2ペルソナ、§5 原則6「登録は後」）/ `UX_DIRECTION.md`
> **作成**: 2026-06-29 / 起案: Claude（オーナー指示によるペルソナ壁打ち）/ ステータス: 🟡 ドラフト（オーナーレビュー待ち）

---

## 0. 背景・問題意識

numanie のシェアカードは「SNS / LINE で共有 → 友人（非ユーザー）がリンクを踏む」という、**新規ユーザー獲得の最上流タッチポイント**である。ところが現状の social UI は獲得を後押しできていない。

### 現状の実装（2026-06-29 時点）

| シェア種別 | URL | social UI |
|-----------|-----|-----------|
| Tier 表シェア | `/share/[shareId]` | **リアクション4種**（いいね/わかる/気になる/見たい）＋ **コメント** ＋ 弱い CTA「numanie を使ってみる →」 |
| 期まとめシェア | `/share/season/[shareId]` | シェア主のキャプションのみ。閲覧者リアクション/コメント無し ＋ CTA「numanie で作る」 |

### 問題点

1. **リアクションもコメントも Google ログイン必須**（`signIn("google")` 直行）。非ユーザーが「いいね」を押すためだけに Google ログインを要求される＝摩擦だけ高く、ほぼ誰も押さない。原則6「登録は後／価値を見せてからログイン」に反する。
2. **リアクション4種が見出しを占有**しつつ、シグナルとしては弱い（ログインゲートで死んでいる）。獲得の主役であるべき「自分も作る」導線が埋もれている。
3. **CTA が弱い・受け身**（「使ってみる →」）。「あなた **も** これを作れる」という自分ごと化が弱い。

## 1. ペルソナ壁打ち

PRODUCT_CONCEPT §3 の2ペルソナの視点で、シェアページ上の各 social 要素を評価する。

### ペルソナA「つん」（ライト層・20代後半・今期2〜3本）

- **シェアページでの立場**: ほぼ常に **閲覧者**（友人の LINE/X リンクを踏んだ非ユーザー）。
- **リアクション**: タップしてみたいが「いいねにログイン要求」で離脱。仮にログイン不要でも、つんにとって他人の Tier 表に「わかる/気になる」と4種から選ぶのは粒度が細かすぎ＝**ほぼ価値ゼロ・ノイズ**。
- **コメント**: 自分から書く可能性は低い（lurker）。ただし「どれが面白い？」を読む価値はある＝**読めれば十分、書く導線は二次**。
- **CTA「自分のカードを作る」**: Tier 表を作るのは重い。だが「今期見てるリスト」程度の軽い共有なら作るかも。**摩擦最小（ログイン前にプレビュー/お試し）でないと作らない**。

### ペルソナB「ぬま」（ヘビー層・30代・今期10本以上・SNSで語りたい）

- **シェアページでの立場**: **シェア主（作り手）** であり、他のぬまのカードの **閲覧者** でもある。
- **リアクション（作り手として）**: 反応＝布教の手応え（ドーパミン/社会的証明）。欲しい気持ちはある。**ただしログインゲートでほぼ付かない＝手応えとして機能していない**。付くなら嬉しいが、4種は過剰で「いいね」1種で足りる。
- **コメント（作り手・閲覧者として）**: ぬま↔ぬまの**議論・感想交換に実価値**。「これ◯話まで見て」「同意」など。ログイン摩擦も「語りたい」層なら許容範囲。**comments こそ残す価値がある engagement**。
- **CTA**: 既に作り手なので押す。文脈に合った遷移先（Tier 表を見た→Tier を作る）だと自然。

### 壁打ち結論（要点）

| 要素 | つん（閲覧） | ぬま（作り手/閲覧） | 判定 |
|------|------------|---------------------|------|
| リアクション4種 | ノイズ・ログイン摩擦 | 欲しいが死んでいる・4種は過剰 | **撤去**（ログインゲート vanity metric。獲得面を散らかすだけ） |
| コメント | 読む価値あり・書くのは稀 | 議論の実価値あり | **残す**（読み公開・書きのみログイン。ただし主役にはしない） |
| 「自分のカードを作る」CTA | 摩擦最小なら可能性 | 自然に押す | **主役に昇格**（コンバージョンの hero。文脈連動・登録前プレビュー） |

→ オーナーの直感（「いいね不要・コメントのみ」）を **ペルソナ分析が支持**。加えて「CTA をコンバージョンの主役に昇格」を本仕様の核とする。

## 2. 仕様（提案）

### 2.1 リアクション撤去

- `/share/[shareId]`（`share-page-client.tsx`）の **リアクション4種ボタン群（`reactionOptions` / `handleReaction` / ヘッダー `share-actions`）を UI から撤去**。
- 閲覧時のリアクション取得（`initialViewerReactions` / reactionCounts 表示）も撤去。
- **データ層は破壊しない**（低リスク優先）: `reactions` テーブル・`/api/shares/[shareId]/reactions` ルート・`lib/shares.ts` の `ReactionKind`/`ReactionCounts` 型は**当面そのまま残す**（dead だが無害）。将来クリーンアップは別 Issue。
  - ※ `BoardShare.reactionCounts` を参照しなくなるだけ。API を消すと既存共有データ取得経路に波及するため今回は触らない。

### 2.2 コメント（維持・微調整）

- コメント機能は維持（`comment-panel`）。**読みは公開のまま、書きのみログイン**（現状維持）。
- 位置づけを「主役」から「二次セクション」に。CTA の下、または折りたたみでも可（実装は最小変更で可）。
- 日本語 UI に統一（現状 "Comments" / "Post" / "No comments yet." 等が英語）。例: 見出し「コメント」、プレースホルダ「感想を書く…」、ボタン「投稿」、空「まだコメントはありません」、未ログイン「Google ログインしてコメント」。
- **期まとめシェアへのコメント追加は今回スコープ外**（期まとめは純粋な獲得面として CTA に集中。コメントは Tier 表シェアのぬま↔ぬま議論に限定）。

### 2.3 「あなたのシェアカードを作る」CTA（核 / 新規 hero）

- **共通部品 `components/ShareCardCTA.tsx` を新設**し、Tier 表シェア・期まとめシェアの両方で使う（重複 CTA の DRY 化）。
- **デザイン**: 獲得の主役にふさわしい目立つブロック（emphasis ボタン＋短い価値コピー）。モバイルでは下部に sticky 表示も検討（実装は段階的、まずは通常ブロックで可）。
- **コピー（自分ごと化・価値先行）**:
  - Tier 表シェア: 「あなたも今期のアニメを Tier 表にして布教しよう」→ ボタン「自分の Tier 表を作る」
  - 期まとめシェア: 「あなたも今期の沼をまとめて共有しよう」→ ボタン「自分のシェアカードを作る」
- **遷移先（文脈連動）**: Tier 表シェア → `/tier?from=share` / 期まとめシェア → 期まとめ作成導線（`/lab/promote` または該当の作成入口）`?from=season-share`。`from` クエリは将来の計測用マーカー（実装は付けるだけで集計は別途）。
- **摩擦最小（原則6）**: 遷移先は **ログイン前に閲覧/お試しできる**こと（既存のゲスト閲覧を踏襲。新規にログインウォールを足さない）。

### 2.4 スコープ外（将来 Issue 候補）

- リアクション API/テーブルの物理削除（cleanup）。
- 期まとめシェアへのコメント機能。
- 匿名コメント（ログイン不要化）→ モデレーション設計が必要なため今回は見送り。
- CTA の sticky bar / 計測イベント集計（収益化90日プランの計測基盤と合流）。
