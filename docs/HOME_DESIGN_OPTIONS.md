## 1. 結論サマリ（最重要）

> **ホームの主役はモードで真逆。1つのホームで両取りは不可能 → モードで出し分ける。**

ペルソナ「つん（ライト）」と「ぬま（ヘビー）」の壁打ちで、同じ機能が**一方には神／他方には邪魔**と判明：

| | つん（シンプル） | ぬま（プロ） |
|---|---|---|
| コア体験1位 | (a) 今夜何見る即決 | (b) 記録・整理・評価 |
| ホーム最上部に欲しい | 今夜見られる作品 | 今週の視聴進捗（未視聴最新話） |
| 記録・評価 | 「正直いらない」 | 「来る動機の80%」 |
| 今夜何見る候補提示 | 「一番嬉しい」 | 「アルゴリズムに決められたくない。邪魔」 |

→ 「シンプル/プロのモード切替で主役を変える」設計が**正しい**ことの裏付け。

### 両ペルソナが一致した点（モード共通の土台）
1. **「今夜放映／最新話あり」をホーム上部に**（つん＝今夜見る／ぬま＝最新話チェック）
2. **下部ナビは1番目ホーム・最後が探す**（起点と発見の位置は共通）
3. **視聴リズム設定（毎週/まとめて/ゆっくり）は死にフィールド**（両者とも不要と明言）→ 廃止候補
4. **PWAバッジは「今日の本数」より「未視聴最新話の数」**にすべき（ぬま）
5. **サブスク最適化は独立タブにするほどではない**（分析/探索にネスト）

## 1.5 視聴スタイル軸（リアタイ/サブスク）— 2026-06-13 追加 ⭐

ペルソナ（ライト/ヘビー）とは**独立した軸**。オーナー指摘で判明し、**ホームの主役選びを左右する最重要の修正**。

| | リアタイ派 | サブスク派 |
|---|---|---|
| いつ見る | 放送日その日（TV） | 放送日の**翌日以降**、自分の都合で（配信） |
| 「今夜放送」 | ✅ 価値高い | ❌ 薄い（その日には見ない） |
| 欲しい情報 | 今夜の放送 | **今 配信で見られる未視聴話** / いつ配信されるか |

### 2×2（ペルソナ × 視聴スタイル）
```
              リアタイ派           サブスク派
ライト(つん)   今夜TVで見る(少)     配信で溜め見(多?)
ヘビー(ぬま)   同時配信リアタイ     配信で計画的に消化
```

### 設計への含意（4案の見直し）
- ✅ **確定（2026-06-14, オーナー）: サブスク派が主軸**。リアタイ派は副次対応。
- ペルソナ議論でも つん・ぬま とも**サブスク派的**で、「今夜放送をその日に」を強く求めた人はいなかった。
- ❌ **「今夜放映」を最上部・大きく出す案（S2 ヒーロー）はリアタイ派限定で外れやすい**。
- ✅ **ホームの主役は「今夜放映」→「配信済み・未視聴のキャッチアップ（今 見られる溜まり）」に寄せる**。
- 今夜放送はリアタイ派向けに**残すが副次**（小さく/オプション）。
- データ面: 「配信済み・未視聴」を正確に出すには視聴済み話数が要る（§3 の地雷と同根）。当面は `airing.nextEpisode` で「最新話が来た」近似で代替可。

## 2. ホーム4案（HTMLモック: `Downloads/numanie-home-mockups.html`）

### シンプルモード（つん向け）— 主役「今/今週どうするか」
- **S1 視聴サポート積み上げ型**: 🌙今夜見られる → 📅今週まだ見てない（未視聴バッジ）→ 🔖見たい（見放題ラベル）。つんの理想を忠実に。
- **S2 今夜の1本ヒーロー型**: 本命1本を大カード＋「▶続きを見る」CTA。即決を最大化。下に今週の残り。

ナビ案: **ホーム / カレンダー / 視聴中 / さがす**

### プロモード（ぬま向け）— 主役「記録・進捗・評価」
- **P1 視聴進捗ダッシュ型**: 今期の未視聴◯話を進捗バー → 🆕最新話あり → ✍️最近の記録 → 🏆マイTier。
- **P2 記録フィード主役型（Letterboxd）**: 自分の評価フィード（星＋メモ）が主役。上に未視聴サマリー薄く、下に「年間ベスト」振り返り入口。

ナビ案: **ホーム / Tier / 分析 / 探索**

### モックに込めた共通言語
- 配信サービスチップ（「dアニメで見られる」一目表示）← つん要望
- **赤の未視聴バッジ**（今日の本数より未視聴最新話）← 両者一致
- 年間ベスト Wrapped 風入口 ← ぬま要望
- プロ版はアルゴリズム提示を排し、自分の進捗/記録を主役に ← ぬま要望

## 3. 改修容易性 調査結果（どの案でも作れるか）

| 案 | 難易度 | 状況 |
|----|--------|------|
| **S1** | 低 | 現状とほぼ同型。すぐ作れる |
| **S2** | 低〜中 | `selectTonightCandidates`（既存）流用可。ヒーロー大カードUIだけ新規 |
| **P2** | 中 | ✅ 星(`favoriteLevel`)・メモ(`notes`)・更新日時(`updatedAt`)が**揃っている**。フィード専用カードの新規実装のみ |
| **P1** | ⚠️**高** | ❌ **「何話まで見たか」を保存するDBフィールドが存在しない** |

### 🔴 P1 の地雷：視聴済み話数が無い
P1の核（「未視聴◯話」進捗バー）は、ユーザーの視聴済み話数データが現状ゼロ。実装には：
- `user_anime_statuses` に `watched_episodes` カラム追加（`ensureStatusSchema` の ALTER）
- `AnimeStatusRecord` 型・`updateTrackingDetails` 拡張
- watchlist に「何話まで見た」入力UI追加
- 工数 半日〜1日。**P1 を選ぶ場合のみ必要**（S1/S2/P2 は影響なし）。

### データ可用性まとめ
| 項目 | 状態 | 根拠 |
|------|------|------|
| a) 視聴済み話数/進捗 | ✗ なし | `lib/statuses.ts` スキーマに該当列なし |
| b) 最新話番号/次回話 | △ 部分的 | `airing.nextEpisode`（AniListスナップショット） |
| c) 配信サービス | ○ あり | `streamingProvidersJp`（enrich済みレコードのみ） |
| d) 評価・メモ・更新日時 | ○ あり | `favoriteLevel`/`notes`/`updatedAt`、`updated_at DESC` で取得済み |

## 4. 実装済みの共通土台（2026-06-13）

「どの案に決まっても無駄にならない」低リスクな土台のみ先行実装済み：

1. **`components/AnimeList.tsx`**: 型を `CardLane` の `LaneCardData` から独立（`AnimeListItem`）。`meta` フィールド（「#7 配信中」「dアニメで見放題」等の行補足）をオプション追加。全案の主力部品。
2. **ホームのモード分岐骨組み**:
   - `app/home-client.tsx` … `useUiMode()` で `simple`/`pro` を出し分けるディスパッチャ
   - `app/home-simple.tsx` … シンプル版（現状の今夜/視聴中/見たい 縦リスト）＋共通の空状態ガイド
   - `app/home-pro.tsx` … プレースホルダ（当面シンプル版を表示。案 P1/P2 確定後に差し替え）
3. **`app/globals.css`**: `.anime-list` / `.anime-row` 系スタイル追加（縦リスト行の見た目）。
4. 横スクロール廃止（`CardLane`→`AnimeList`）はホーム・視聴中カレンダーに適用済み。

→ 案が S/P いずれに決まっても、`home-simple.tsx` か `home-pro.tsx` の中身を差し込むだけ。`home-client.tsx`（分岐）は変更不要。
