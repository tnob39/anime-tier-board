## 5. 認証別ナビ・閲覧・保存・復帰（現行）

ロール定義:

| ロール | 定義 |
|--------|------|
| **ゲスト** | 未ログイン |
| **一般ログインユーザー** | Google ログイン済みかつオーナーメール以外 |
| **オーナー** | ログイン済みかつ `isOwnerEmail` 該当 |

### 5.1 表示（ボトムナビ）— リリース目標（Nav V5 有効時）

> **実装状態**: 現行実装は feature flag により legacy nav が既定であり、この表は #417 のリリース目標である。flag 未有効の状態を「現行UI」とは扱わない。

| ロール | リリース目標のナビ表示 |
|--------|------------------------|
| ゲスト | **5 タブの構造を維持**（即ログイン壁なし）。ただし `さがす` は `ownerOnly` のため **非表示** → 表示は ホーム / Tier / マイリスト / マイページ |
| 一般ログインユーザー | 同上（`さがす` 非表示）。表示は 4 本 |
| オーナー | **5 タブすべて表示**（ホーム / Tier / さがす / マイリスト / マイページ） |

> Visual / Simple はナビ本数・ラベル・順序を変えない。

### 5.2 閲覧

| 領域 | ゲスト | 一般ログイン | オーナー |
|------|--------|--------------|----------|
| ホーム `/` · Tier `/tier` | 閲覧可（ゲスト向けホーム / ローカル board） | 可 | 可 |
| 共有ページ `/share/...` 等 | **読み取り可** | 可 | 可 |
| マイページ `/mypage` | **枠は閲覧可**（ログイン CTA・公開リンク集）。個人サマリーは空/誘導 | 自分のデータ表示 | 可 |
| マイリスト `/watchlist` | ページゲート: `/?login=required` | 可 | 可 |
| 分析 `/dashboard`（マイページ経由） | ページゲート: `/?login=required` | 可 | 可 |
| 設定 `/settings` · 声優 `/voice-actors` | ページゲート: `/?login=required` | 可 | 可 |
| さがす `/explore` | 不可（`/` へ） | 不可（`/` へ） | **可** |

### 5.3 保存時ログイン

| 操作 | ゲスト | ログイン後（一般 / オーナー） |
|------|--------|------------------------------|
| Tier のローカル編集 | **可**（価値を先に見せる） | 可 |
| クラウド保存・同期 | **保存時に Google ログイン要求** | 可 |
| 視聴ステータス / マイリスト更新（サーバ） | **保存時にログイン要求** | 可 |
| 共有のコメント投稿 | **書き込み時にログイン要求**（読みは公開） | 可 |
| 共有 URL 作成・クラウド共有 | ログイン要求 | 可 |

原則: **登録は後**。閲覧・試用の前に全面ログイン壁を置かない。永続化・ソーシャル書き込みの瞬間にログインする。

### 5.4 認証後復帰 — リリース必須契約（**未実装**）

> **実装状態**: 現行は `/?login=required` の通知までで、return URL と保存操作の再開は未実装。この表は後続の P0 原子タスクが満たすべき契約であり、完了済み挙動として扱わない。

| 起点 | リリース時に満たす復帰先 |
|------|--------------------------|
| 保護ページ到達（マイリスト / 分析 / 設定 / 声優） | ログイン完了後、**元の保護ページへ戻す**（ゲートは `/?login=required` 等で意図を示す） |
| 保存・投稿 CTA からの `signIn` | ログイン完了後、**元画面に戻り同一操作を再開できる状態**にする |
| マイページ / ホームの「Google でログイン」 | ログイン完了後、**起動した画面（マイページまたはホーム）に留まる** |
| さがす（非オーナー） | ログインしても一般ユーザーは到達不可。オーナーのみ表示・到達 |

## 6. Visual / Simple（同一機能の表示モード）

Visual / Simple は **表示設定のみ**。次を満たす。

| 規則 | 内容 |
|------|------|
| 同一 IA | タブ構成・ルート・Bottom Nav 外導線は共通 |
| 同一到達機能 | どちらのモードでも同じ画面・操作・API 結果に到達できる |
| 同一主要 CTA | 主要 CTA を片方のモードだけで隠さない・別ラベルの別機能にしない |
| Simple 画像 | **作品画像の HTTP リクエストを出さない**。**空の画像プレースホルダ領域を残さない**（縦リスト等へ再構成） |
| Visual 画像 | 画像を識別補助として表示。カード全体を操作対象にする |

**禁止**: Visual / Simple を simple/pro 時代のような **別アプリ IA** や hidden CTA の出し分けに使うこと。

## 7. text-first / アイコン / Tier 色（現行）

| 規則 | 内容 |
|------|------|
| text-first | UI の意味は **常時可視のテキスト**で伝える |
| 単独アイコン禁止 | 意味を担うコントロールを **アイコンのみ**にしない |
| 装飾アイコン禁止 | 意味を持たない装飾だけのアイコンを置かない |
| utility icon | 残す場合は **常時可視ラベルを併記**する（例: ボトムナビはアイコン + ラベル） |
| aria の位置づけ | `aria-label` / tooltip は **補助**。可視ラベルの **代替にしない** |
| Tier 色 | **データ色**。ランク識別以外の装飾・無関係ボタン・アラートに流用しない |
| Tier アイコン / ラベル | 色だけに依存しない。短いテキスト（S/A/B 等）または同等の可視ラベルを伴う |
| 反応・ステータス | 色変化だけで状態を伝えない（枠・塗り・テキスト等を併用） |

ボトムナビは現行どおり **アイコン + ラベル**。ラベル省略や「選択中のみラベル」は text-first に反するため採用しない。

## 8. 実装ガードレール（必守）

実装・レビュー・委任時に次を守る。違反しそうなら先に本記録と関連 SSOT を更新して合意する。

1. **底部ナビを 4 タブや simple/pro 別タブに戻さない。** 正規は §2 の順序付き 5 タブ。
2. **`/dashboard` を MobileNav に載せない。** 到達はマイページ（または同等の集約導線）から。
3. **Visual / Simple は表示設定のみ。** IA・ルート・到達機能・主要 CTA を変えない（§6）。
4. **text-first を守る。** 単独・装飾アイコン禁止。utility icon は常時可視ラベル併記（§7）。
5. **認証別規則は §5 に従う。** ゲストに全面ログイン壁を足さない。保存時ログイン・認証後復帰を崩さない。
6. **旧ドキュメントの「現行」「最優先」表現に引きずられない。** 方針③・N1–N4・4 タブ・simple/pro IA・native 案A 推奨は **履歴**。
7. **方針④ §6 の「バックエンド無変更」を Home / progress API に拡張適用しない。**
8. **存在しない SSOT ファイルを canonical と称さない。** 未コミット・未作成の screen spec / backend contract を「既に正本」と書かない。
9. **日本語 UI 文言は UTF-8。** 文字化け（mojibake）をコミットしない。
10. **新 SSOT ファイルを増やさない。** 本主題の正本分担は §9。
